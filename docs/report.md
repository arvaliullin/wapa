**Слайд 2. Актуальность темы**

Современные веб-технологии предъявляют высокие требования к скорости работы приложений.  
JavaScript - основной язык фронтенда - заметно уступает по производительности нативным решениям при вычислительно сложных задачах.  
Появление WebAssembly позволяет встраивать в веб-приложения модули, написанные на более эффективных языках - C++, Rust и Go.
Однако остаётся неясным: какой язык лучше выбрать для WebAssembly, как интегрировать эти модули и какой стек разработки будет самым эффективным.

Однако нет унифицированной методики опытной оценки на различных аппаратных платформах, отсутствуют устоявшиеся рекомендации по выбору стека для различных сценариев.

**Слайд 11. Проектирование платформы**

Для экспериментов была спроектирована микросервисная программная платформа.  
В основе: 
- база данных PostgreSQL для хранения результатов,
- брокер сообщений NATS для обмена задачами между сервисами,
- сервис по управленю замерами,
- сервис по запуску экспериментов.
  
Весь стек написан на Go, задействуется контейнеризация Docker.  
Для точного и автоматического измерения времени исполнения используется утилита Hyperfine.

---

**Слайд 12. Сервис по управлению замерами**

Ключевой компонент платформы - сервис по управлению замерами.

**Слайд 13. Сервисы запуска экспериментов**

Сервисы запуска получают задание через очередь NATS, запускают hyperfine с нужными параметрами и отдают данные обратно сервису управления замерами.
